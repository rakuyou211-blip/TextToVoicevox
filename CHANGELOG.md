# 変更履歴

日付は日本時間。バージョンはアプリのタイトルバー（`テキスト抽出 → VOICEVOX vX.Y.Z`）でも確認できます。

## v1.18.0 — 2026-07-18（立ち絵が全キャラ対応・試聴の無反応対策）

**立ち絵が全43キャラ対応に**
- これまで立ち絵は **ずんだもん／四国めたん の2キャラだけ**でしたが、VOICEVOXの**全キャラ**に対応しました。`assets/立ち絵/` に **`キャラ名.png`**（例 `春日部つむぎ.png`・`九州そら.png`）を置くだけで、その話者を選んだときに立ち絵が出ます（まばたき・口パクの差分 `_closed`/`_open`/`_blink` も同じ命名で対応）
- **選んだ話者の立ち絵が無いときは、別キャラを誤って出さず枠を空に**します（連続再生で話者が変わっても、立ち絵のあるキャラだけ正しく表示・口パク）
- **ファイル名に使えない文字を含むキャラも自動で照合**（`小夜/SAYO`→`小夜SAYO.png`、`†聖騎士 紅桜†`→空白を除いて保存）。旧 `zundamon.png`/`metan.png`（ローマ字）もそのまま使えます
- 立ち絵はキャラごとに権利者が違うため、画面のクレジットを汎用表記に変更。各キャラのガイドラインの確認方法・置き方は [`assets/立ち絵/README.md`](assets/立ち絵/README.md) にまとめました（**立ち絵は引き続き同梱しません**）

**試聴まわりの堅牢化**
- 試聴・連続再生を押しても無反応に見えるケースを解消：再生中に押したら理由を表示、未接続なら「VOICEVOXを起動してね」の具体案内、■停止で状態が戻らないときの強制復帰を追加

## v1.17.0 — 2026-07-18（10時間の本に耐える・途中でやめても無駄にならない）

**長編対応（今回の目玉）**
- **メモリがほぼ一定に**：音声生成が全行のWAVをメモリに溜めてから結合する方式（10時間級の本で数GB）から、行ごとにディスクへ書いて逐次結合する方式（常に数MB）に。**24時間超の全巻結合はWAVの4GB上限を保存前に検知**して分割かM4A/M4Bを案内します（従来は全行合成後の保存で全損でした）
- **キャンセルしても「ここまでを保存」**：全文結合の生成を途中でやめたとき、合成済みの部分を「◯◯_途中まで.m4a」として保存できます（章・字幕付き。従来は成果ゼロでした）
- **1行の失敗で全巻が全損しない**：エンジンの一時不調は行ごとに2回まで自動リトライ。連続再生は失敗行をスキップして読み続けます（連続3失敗＝エンジン停止のときだけ従来どおりエラー）
- **キャッシュが長編で効くように**：生成中に自分の前半行を追い出す自己破壊を防止（実行中エントリの保護）。上限も変更可能に（§4「キャッシュ…」から。10時間級を常用なら2000MB推奨）
- **OCRの途中キャンセルとページ進捗**：数百ページのスキャンPDFでもキャンセルが数秒で効き、「OCR実行中... (37/300枚)」と実際の進捗が見えます（従来はOCR全体が終わるまで待たされました）。外部プロセス（Windows OCR・音声変換）にはタイムアウトを設定し、固まってもUIが永久busyにならないように

**新機能**
- **📄 PDFのページ範囲指定**：§2「ページ:」欄（例 `5-320` / `1-3,10-`）で表紙・目次・索引を読み飛ばせます（CLIは `--pages`。全角数字・「、」「〜」もOK）
- **🗂 セリフ一覧CSV**：分割出力時に「セリフ一覧.csv」（ファイル名・話者・テキスト・長さ秒）を自動保存。動画編集ソフトでの素材整理の対応表に（Excel対応のBOM付きUTF-8）
- **🏷 掛け合いのファイル名に話者名**：複数話者のとき `001_ずんだもん_こんにちは.wav` 形式に（ファイル名だけで誰の声か分かる）
- **✅ @話者タグの事前検証**：タイプミスで解決できないタグを合成前に指摘し、該当行へジャンプできます（従来は全編を聴き直すまで気づけませんでした）
- **🧹 キャッシュ管理**：§4接続行の「キャッシュ…」から容量確認・上限変更・クリア・フォルダを開くが1画面で
- **📊 行数・文字数・読み上げめやすの常時表示**（§3右上。＃メモ行・@タグは除いて計算）
- **CLIにも合成キャッシュ**（`--no-cache` で無効）。台本を1行直して再実行→未変更行は即完了

**直したところ**
- 分割出力が999ファイルを超えると連番が4桁になり名前順ソートが崩れていた（総数に応じた桁数に）
- 数値欄が空のまま「プリセット保存」すると無反応で保存されなかった
- M4B選択時に「N行」欄が編集できたが値は無視されていた（灰色に）
- 辞書ハッシュの取得失敗時、二度と使われないキャッシュファイルを書き続けていた（キャッシュ汚染）
- 試聴・声サンプル再生中に「⏵ 続きから」が押せて画面だけ飛んでいた
- READMEの版数が4リリース前（v1.11.0）のまま取り残されていた → **版数とドキュメントの整合をテストで強制**（tests/test_docs.py 新設。今後は更新漏れのままコミットするとテストが落ちます）。英語READMEにM4B・声サンプル・読み確認等の欠落機能を追記し、最低Pythonバージョン表記を3.9に統一

**内部の見直し**
- M4Bチャプター構築・出力グループ化・OCR補正をGUI/CLI共有のcore関数へ統合（片方だけ実装される事故の防止）
- メモ行判定の残っていたコピペ2箇所を core.is_memo_line に統一

**リリース前レビューで見つけた取りこぼしの修正**
- 音声生成のスプール作成がディスクフルで失敗すると、操作不能（永久busy）になっていた → エラー表示して復帰するように
- キャッシュ上限を下げて「適用」しても超過分が削除されず表示も矛盾していた → 即座に削除し、起動時にも前回の超過分を掃除
- @話者タグ確認の「いいえ」で飛ぶ行が、本文先頭に空行があるとずれていた → 正しい行へジャンプ
- キャンセル時の部分保存で連結に失敗すると書きかけの `.part` が残っていた → 残さないように

## v1.16.0 — 2026-07-18（声選びが速く・直した行だけ再合成・壊れない保存）

**新機能**
- **🗣 話者選択が2段に**：170超のスタイルが1本に並んでいたコンボを「キャラ → スタイル」の2段に分割。キャラを選んで「🔊 声を聴く」でスタイルを聴き比べる、が自然にできます
- **⚡ 行単位の合成キャッシュ**：同じ行・同じ声・同じ設定の合成結果を自動で再利用（上限500MB・古い順に自動削除）。**誤読を1行直して再生成→他の行は即完了**、連続再生で聴いた本の音声生成もほぼ一瞬に。辞書やエンジンを変えると自動で作り直します
- **⛔ テキスト抽出のキャンセル**：数百ページのスキャンPDFのOCRを途中で止められます（抽出中はボタンが「⛔ キャンセル」に。Escでも可・そこまでの部分結果は表示）
- **■ 試聴・声サンプルも止められる**：長い行の試聴を「■ 停止」/Escで途中停止できます（従来は最後まで待つしかありませんでした）
- **📖 しおりの可視化**：「⏵ 続きから」で再開する行が淡紫色でマークされ、どこまで聴いたかが一目で分かります
- **📚 M4B: 章見出しが無い本にも自動チャプター**（約10分ごと・区切り行の本文冒頭がタイトルに）。Apple Books等で長い本の頭出しができます
- **📕 読み方辞書の強化**:「追加/上書き」（同じ単語は重複せず上書き）・一覧ダブルクリックで編集・「▶ 読みを試聴」で登録した読みをその場で確認
- **🖱 右クリックに「元に戻す/やり直す」**（WindowsはCtrl+Yのやり直しも）
- **⌨ 入力ファイルのキーボード操作**：Delete=削除・Alt+↑↓=並べ替え・選択でフルパス表示（同名ファイルの区別に）

**CLI**
- **--unit each/combine/nlines/para**（GUIの「まとめ方」と同等。--split-lines N・--keep-blank も）と、分割出力でも --srt でファイルごとに字幕保存
- **--vvproj**：台本txtからVOICEVOXプロジェクトを一括生成（行別@話者タグ対応）
- **合成が3並列に**（GUIと同じ。長編で約3倍速）・**--no-preprocess**・**--name-snippet**（001_本文冒頭.wav をオプトインで）・末尾スラッシュ付きURLでも接続できるように

**テキストの精度**
- **《》の誤削除を修正**：なろう/カクヨム系の「彼は《ファイアボール》を放った」のスキル名・強調が消えていたのを、ルビ（漢字直後・｜指定）だけ削除するように限定。**カクヨム傍点《《…》》も正しく本文へ展開**（従来は本文が消えて「》」が残る破損でした）
- **ノイズ除去をOCR由来テキスト限定に**：小説txt中の英文の引用行・年号だけの行が「画面キャプチャのノイズ」として消えていたのを修正（txt/docx/EPUB・PDFテキスト層には触れません）
- **PDF「自動」の改善**：透かし（Scanned by …）やページ番号だけのテキスト層があるスキャンPDFで、本文が丸ごと失われていたのをOCRと比較して良い方を採用するように
- **Word（.docx）のテキストボックス**：同じ文言が最大4回読み上げられていた重複抽出を修正
- **EPUB**：目次ページ（nav）と linear="no" の付録を本文から除外（全章が対象になる誤ラベルEPUBは従来どおり全文）
- **EUC-JPのテキストに対応**：文字コード判定をスコア方式にし、古い配布txtの文字化け読み上げを解消
- **半角カナ→全角**：ｶﾞｲﾄﾞ→ガイド・半角句点｡でも文分割が効くように（「全角英数→半角」ON時）
- **数値レンジ「10〜20」→「10から20」**・組文字単位の読み展開（㌔㌢㍉㌘㌧㌫㌍㌦㍗・㍾㍽㍼㍻㋿）

**直したところ（設定・データを守る修正）**
- **設定消失バグ3件を修正**：数値欄（話速・DPI等）を空にしたまま閉じると settings.json が0バイトに壊れて全設定（プリセット・置換ルール・テーマ・しおり）が消えていた／エンジン未接続で閉じると話者選択が空文字で上書きされていた／どちらも空欄は既定値へフォールバックし、保存はアトミック書き込み（書き込み途中のクラッシュ・電源断でも壊れない）に。壊れた設定は .bak に退避
- **UI永久ロック2件を修正**：数値欄が空のまま「▶ 試聴」「▶▶ 連続再生」を押すと全操作が拒否され再起動しかなくなっていた
- **UIフリーズの単一障害点を解消**：内部メッセージ処理の例外でUI全体が固まる構造を、1件落として続行する方式に
- 生成中の Ctrl/Cmd+G が無警告で生成をキャンセルしていた（生成中は無効に。キャンセルはEsc/ボタンで明示的に）
- 選択範囲内の右クリックで「この行を試聴」「@タグ挿入」がクリックした行と別の行に効いていた
- ＃メモ行トグルが選択終端の行頭（未選択の行）まで巻き込んでいた
- 検索で最初に「↑前」を押すと末尾でなく末尾から2番目に飛んでいた
- 辞書JSONの読み込みでファイル内の重複単語がそのまま重複登録されていた
- パレットPNG/GIFの拡大縮小が低品質（ニアレストネイバー）でOCR精度を下げていた
- 一括置換・全ルール適用で表示が先頭へ飛んでいた（カーソル・スクロール位置を保持）
- VOICEVOX起動待ちの自動接続でボタンが3秒ごとに明滅していた（静かに確認する方式に）
- 実行中にウィンドウを閉じると書きかけのファイルが壊れたまま残っていた（確認ダイアログを追加）
- 分割出力（001_○○.wav）と結合時SRTが無確認で上書きされていた（事前に件数を確認）

**見直し・削除**
- 「N行」「文間の無音」の欄は、効かないまとめ方のときは灰色に（触れるのに無視される欄を排除）
- 試聴の内部往復を削減（audio_query 1回で合成と読み確認を両方）
- 未使用コードの削除（denoise内部の不使用関数・OCRヘルパーの不要引数・重複した色設定）

## v1.14.0 — 2026-07-18（誤読に気づける・写真に強くなる）

**新機能**
- **🔍 読み確認**：`▶ 試聴` すると「読み『ダイ イッショオ、コンニチワ…』」と**VOICEVOXが実際にどう読んだか**をカナで表示。誤読（例: O157→オオイチゴオナナ）に気づいたら「読み方辞書...」で直せます
- **⏱ 読み上げ時間のめやす**：抽出完了時に「読み上げめやす12分」のように所要時間の概算を表示（話速設定を反映）
- **＃メモ行**：行頭に `#` を付けた行は読み上げ・音声生成・vvproj出力からスキップ。台本のメモや指示書きに使えます
- **URL・メールを読み飛ばす（既定ON）**：読み上げると1文字ずつ読まれてしまうURL・メールアドレスを本文から除去（詳細設定でOFF可）

**画像認識の精度 第3弾**
- **スマホ写真の向きを自動補正**：EXIFの回転情報を反映（今までは横向き写真がほぼ読めないことがありました）
- **回転リトライ**：ほぼ読めなかった画像は90°/270°回転でも再OCRし、一番読めた結果を採用（横倒しスキャン・写真対策。macOS）
- **PDFページも自動再OCRの対象に**：影・ムラのあるページはページを描画し直してリトライ（今までは単体画像のみ）

**テキストの精度**
- 数字の桁区切りを除去：`1,234円`→`1234円`（「いち、にさんよん」と誤読されるため。「全角英数→半角」ON時）
- 中黒の連続 `・・・` → `…` に統一（同上）
- OCR誤字補正に `ハ→八` を追加（`十ハ番`→`十八番`。「ハイテク」等の実在語は保護）

**CLI**
- `--format m4b`（章付きオーディオブック）・`--strip-urls`（既定ON）を追加。＃メモ行・クレジット表記の案内もGUIと同様に対応

**見直し**
- M4B選択中は「まとめ方」を無効表示に（M4Bは常に全文結合のため）
- `▶ 試聴` 中もいま読んでいる行をハイライト表示（連続再生と同じ見た目に）

## v1.13.0 — 2026-07-18（声を選んで、本として聴ける）

**新機能**
- **🔊 声サンプル試聴**：話者コンボの隣の「🔊 声を聴く」で、選択中キャラの**公式ボイスサンプル**をワンクリック再生。声選びに迷わなくなりました（エンジン同梱の公式音源をアプリ内試聴のみに使用）
- **📚 オーディオブック出力（M4B・章付き）**：音声形式に **M4B** を追加。全文を1ファイルに結合し、本文の章見出し（`第◯章`・`プロローグ` 等）を自動検出して**チャプター**を埋め込みます。Apple Booksやブックプレーヤーアプリで「本」として頭出しできます（純Python実装・AVFoundationでの読み戻し検証済み。章見出しが無い場合は章なしで保存）
- **⛔ 音声生成のキャンセル**：生成中は「🔊 音声を生成」が「⛔ キャンセル」に変わり、長い本の生成を途中で止められます（Escキーでも可）
- **⏸ 連続再生の一時停止/再開**：停止すると頭からだった連続再生に一時停止を追加。**スペースキー**でも切替でき、再開すると同じ行の頭から読み直します
- **📋 整形レポート**：ノイズ除去で消えた行とOCR誤字補正の内容を§3の「レポート」ボタンで確認可能に。誤って消えた本文のコピー復元や、補正のON/OFF判断に使えます

派手な新機能はなし。人に渡しても最初のところで詰まらない・固まらない、を目的にした安定化です。

**直したところ**
- 数値の欄（OCR解像度・話速・音高・抑揚・音量・無音）を空にしたまま実行すると、画面全体が固まって操作できなくなることがあった問題を修正。
- 前回サブモニタで閉じたあと、そのモニタを外して起動すると窓が画面の外に出て「起動しない」ように見えることがあった問題を修正（画面内に戻します）。
- 章のファイル名に日本語や空白を含む EPUB で、章がまるごと読み飛ばされることがあった問題を修正。全章読めなかったときはエラーで気づけるようにしました。
- 「連続再生（カーソル行から）」で、末尾の空行にカーソルがあると先頭から再生し直してしまう挙動を修正。
- 連続再生の最中にテキスト抽出などを始められてしまい、再生中の本文が入れ替わる不整合を防止。
- M4A/MP3 変換に失敗したとき、日本語のフォルダ名が原因でエラー表示自体が化けて落ちることがあったのを修正。

**配布まわり**
- まっさらな Windows（Python 未導入）で `setup.bat` を動かしたとき、Microsoft Store のダミー python に当たって不親切なメッセージで止まっていたのを改善。Python の入れ方（python.org / "Add python.exe to PATH"）を案内します。
- 依存ライブラリを `requirements.txt` に一本化し、バージョンに上限を付けました（将来ライブラリ側の破壊的更新で新規インストールだけ壊れるのを防ぐため）。
- アプリのバージョンをタイトルバー・`cli.py --version` に表示。不具合報告のときに版が分かります。

**後片付け・その他**
- OCR に使った一時ファイル（元文書の画像コピー・抽出テキスト）を、処理後に必ず削除するようにしました（%TEMP% に残らない）。
- CLI: `--srt` を `--combine` なしで指定したときに警告を出すように。`@話者名:` タグを GUI と同じく解釈するように（タグは読み上げず、その行だけ話者を切り替え）。

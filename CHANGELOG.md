# 変更履歴

日付は日本時間。バージョンはアプリのタイトルバー（`テキスト抽出 → VOICEVOX vX.Y.Z`）でも確認できます。

## v1.14.0 — 2026-07-18（誤読に気づける・写真に強くなる）

**新機能**
- **🔍 読み確認**：`▶ 試聴` すると「読み『ダイ イッショオ、コンニチワ…』」と**VOICEVOXが実際にどう読んだか**をカナで表示。誤読（例: O157→オオイチゴオナナ）に気づいたら「読み方辞書...」で直せます
- **⏱ 読み上げ時間のめやす**：抽出完了時に「読み上げめやす12分」のように所要時間の概算を表示（話速設定を反映）
- **＃メモ行**：行頭に `#` を付けた行は読み上げ・音声生成・vvproj出力からスキップ。台本のメモや指示書きに使えます
- **URL・メールを読み飛ばす（既定ON）**：読み上げると1文字ずつ読まれてしまうURL・メールアドレスを本文から除去（詳細設定でOFF可）

**画像認識の精度 第3弾**
- **スマホ写真の向きを自動補正**：EXIFの回転情報を反映（今までは横向き写真がほぼ読めないことがありました）
- **回転リトライ**：ほぼ読めなかった画像は90°/270°回転でも再OCRし、一番読めた結果を採用（横倒しスキャン・写真対策。macOS）
- **PDFページも自動再OCRの対象に**：影・ムラのあるページはページを描画し直してリトライ（今までは単体画像のみ）

**テキストの精度**
- 数字の桁区切りを除去：`1,234円`→`1234円`（「いち、にさんよん」と誤読されるため。「全角英数→半角」ON時）
- 中黒の連続 `・・・` → `…` に統一（同上）
- OCR誤字補正に `ハ→八` を追加（`十ハ番`→`十八番`。「ハイテク」等の実在語は保護）

**CLI**
- `--format m4b`（章付きオーディオブック）・`--strip-urls`（既定ON）を追加。＃メモ行・クレジット表記の案内もGUIと同様に対応

**見直し**
- M4B選択中は「まとめ方」を無効表示に（M4Bは常に全文結合のため）
- `▶ 試聴` 中もいま読んでいる行をハイライト表示（連続再生と同じ見た目に）

## v1.13.0 — 2026-07-18（声を選んで、本として聴ける）

**新機能**
- **🔊 声サンプル試聴**：話者コンボの隣の「🔊 声を聴く」で、選択中キャラの**公式ボイスサンプル**をワンクリック再生。声選びに迷わなくなりました（エンジン同梱の公式音源をアプリ内試聴のみに使用）
- **📚 オーディオブック出力（M4B・章付き）**：音声形式に **M4B** を追加。全文を1ファイルに結合し、本文の章見出し（`第◯章`・`プロローグ` 等）を自動検出して**チャプター**を埋め込みます。Apple Booksやブックプレーヤーアプリで「本」として頭出しできます（純Python実装・AVFoundationでの読み戻し検証済み。章見出しが無い場合は章なしで保存）
- **⛔ 音声生成のキャンセル**：生成中は「🔊 音声を生成」が「⛔ キャンセル」に変わり、長い本の生成を途中で止められます（Escキーでも可）
- **⏸ 連続再生の一時停止/再開**：停止すると頭からだった連続再生に一時停止を追加。**スペースキー**でも切替でき、再開すると同じ行の頭から読み直します
- **📋 整形レポート**：ノイズ除去で消えた行とOCR誤字補正の内容を§3の「レポート」ボタンで確認可能に。誤って消えた本文のコピー復元や、補正のON/OFF判断に使えます

**OCRの精度 第2弾**
- **2段組の読み順を修正**：段組PDFのOCRで左右の行が交互に混ざっていたのを、段組と確信できる場合のみ列単位（左→右）で読むように（誤検出防止のガード付き）
- **写真の自動再OCR**：影や照明ムラでほぼ読めなかった画像は、照明を平坦化して自動でもう一度OCRし、良い方を採用（macOS・低品質時のみ実行で通常は速度影響ゼロ）
- **OCR失敗が見えるように**：失敗理由と「文字を検出できなかった枚数」を抽出後の警告に表示（従来は無言で空になっていました）

**公式規定の遵守を明示**
- 音声生成の完了ダイアログに、公開時に必要な**クレジット表記（`VOICEVOX:話者名`）を毎回表示**。M4Bにも同じ案内を含めます（VOICEVOX利用規約・各キャラクター利用ガイドライン準拠。サンプル音声・立ち絵などの公式素材は引き続き同梱しません）

**UI・人間味**
- **接続まわりをすっきり**：エンジン接続後はURL欄・接続ボタンを畳んで「**● 接続OK (vX)**」（テーマ対応の緑）＋「接続設定…」だけに。失敗時は自動でフル表示に戻ります
- **エラーが人間の言葉に**：生のエラーコードの代わりに「VOICEVOXが起動しているか確認してね」のような**原因のヒントと次の一手**を表示。技術的な詳細は「詳細を表示」を押したときだけ（不具合報告にはこちらを）
- **あいさつと道案内**：起動時に時間帯のあいさつ（おはよう☀️/こんにちは🍵/こんばんは🌙）、初回起動では3ステップの使い方を案内
- **節目のことばを温かく**：「✅ 抽出できたよ」「🎉 音声ができたよ！」「📖 最後まで読み終わったよ、おつかれさま！」。残り2分を超える長い生成には「のんびり待っててね☕」を添えます

## v1.12.0 — 2026-07-18（OCRがかしこく・文が自然に・次に押すボタンが分かる）

**OCRの精度（写真・スクショ・スキャンPDF）**
- **WindowsでもmacOSと同じ“座標つきOCR”に**：`ocr_win.ps1` が行の外接矩形を返すようになり、これまでmacOS限定だった **折り返し連結**（割れた1文のつなぎ直し）と**映像内ラベルの除去**（局ロゴ・番組名・日時・カテゴリ）がWindowsでも効きます（旧形式との互換維持・傾き検出時は安全側で従来動作）。※Windows実機での実画像確認は次回リリースまでに実施予定
- **Windowsの“無言でOCR空振り”を修正**：Windows OCRエンジンの上限（長辺2600px）を超える画像（300dpiのA4スキャン等）が例外→空文字になっていたのを、前処理とPowerShell側の両方で上限内に収めるよう修正
- **前処理の見直し**：透過PNG（ウィンドウ撮影・クリップボード画像）が黒く潰れてOCRを阻害していたのを白背景に合成。macOSではグレースケール強制をやめ**カラーのままApple Visionへ**（NN系はカラーで精度が出る）。Windowsの強調はハローの少ないUnsharpMaskに変更。「画像前処理」OFFが本当に無加工（サイズ上限のみ）になるよう修正
- macOSのVision OCRで日本語対応リビジョン（Revision 3）を明示指定（環境差で古い版に落ちる事故を防止）
- クリップボードOCRの一時画像が `%TEMP%` に残り続けていたのを修正（OCR後に必ず削除）

**テキストの精度**
- **OCR誤字の文脈補正（新設・既定ON）**：`サ一ビス`→`サービス`、`卜ヨタ`→`トヨタ`、`二ュース`→`ニュース`、`入カ完了`→`入力完了`、`2O26年`→`2026年` など、同形文字の取り違えを前後の文字種で確定できるときだけ補正。「数カ月」「口コミ」「口パク」「二ヶ月」「O157」「H2O」などの実在語はガードで保護（迷ったら触らない）。**OCR結果にだけ**適用され、txt・docx・EPUB・PDFテキスト層には触れません（§2「OCR誤字を補正」でOFF可・CLIは `--no-fix-confusables`）
- **「」（）の内側で文を切らないように**：`「もう帰る。」と彼は言った。` が2つに割れて不自然なポーズ・字幕になっていたのを1文のまま合成。`えっ！？` の連続記号も1文に
- **読めない記号の展開**：「全角英数→半角」ONのとき、`①`→`1`・`Ⅲ`→`3`・`㈱`→`株式会社`・`50㎡`→`50平方メートル`・`25℃`→`25度`・`№`→`ナンバー` などを読みに展開（※既にONで使っていた方は挙動が少し変わります）
- **ノイズ除去の改善**：`写真：ロイター` 等のクレジット行と `1.2万人` 等の視聴数オーバーレイを除去対象に追加。逆に日本語記事中の `iPhone 17 Pro Max` のような英数の製品名見出しが誤って消えていたのを保護
- **青空文庫の対応強化**：踊り字（`こゝろ`→`こころ`・`つゞく`→`つづく`）とくの字点（`どき／＼`→`どきどき`）を読める形に展開。末尾の「底本：…」と冒頭の記号説明ブロックを読み上げから除去

**ボタン配置・操作**
- **「次に押すボタン」がひと目で分かる**：本文が空のときは `▶ テキスト抽出 実行`、本文ができたら `🔊 音声を生成` だけがオレンジ（Primary）になります
- **押せないボタンが理由を教える**：無効中の `🔊 音声を生成`・`▶ 試聴`・`プロジェクト保存` などにマウスを乗せると「エンジン未接続です。接続確認を押すと使えます」等の理由をツールチップ表示
- **主要操作のショートカット**：抽出=Ctrl/⌘+Return・音声生成=Ctrl/⌘+G・ファイル追加=Ctrl/⌘+O・txt保存=Ctrl/⌘+S・試聴=Ctrl/⌘+P・停止=Esc（ツールチップにも表記）
- **§3の置換バーを整理**：ルールの「登録/削除」を `ルール ▾` メニューにまとめ、常時グレーだった「復元」を「本文を全消去」と1ボタンに統合（全消去すると「復元」に切り替わる）
- **§4をすっきり**：使わない人も多い「プリセット/セリフ別話者」行を折りたたみ式に（右端の `プリセット/セリフ ▸` で開閉・状態は保存・セリフ別話者ON時は自動で開く）。区切りの余白も拡大
- 立ち絵パネルの自動開閉にヒステリシスとデバウンスを追加（境界幅でウィンドウをドラッグしたときのチラつきを解消）

## v1.11.0 — 2026-07-17（テーマを選べる・押しやすいボタン）

- ライトテーマの文字色・アクセントのコントラストを微調整（見出しや説明文がくっきり）。
- **狭い窓でも大事なものから残る配置に**：テーマ選択・主要ボタンを優先確保し、あふれた時は状態文だけが縮むように。再生ボタンの補足（「カーソル行」等）はラベルからツールチップへ移し、`▶ 試聴`／`▶▶ 連続再生` と短く（最小幅860pxでも全ボタンが判読可能に）。

## v1.7.0 — 2026-07-14（映像内ラベルの除去）

**新機能**
- **映像内ラベルの除去**：ニュース画像OCRに残りがちな**局ロゴ・番組名・日時・カテゴリ表示などの短い日本語ラベル**（例「MBSニュース」「国内」「TBS TBSテレビ」「国際」）を、行の**座標・文字サイズ**を使って除去します。これらは正しい日本語なのでテキストだけでは本文中の短語と区別できませんでした。本文（画面で最も支配的な段落）から外れた「①文字サイズが本文と違う ②画面最上部にある短い行 ③本文の折り返しではない孤立した短い行」のうち**複数条件を満たす行だけ**を落とします。**単一条件では落とさず**、句読点・鉤括弧を含む行／段落ブロックの一部／一定長以上の行は必ず残すため、**本文の取りこぼしは起きません**（迷ったら残す方針）。macOS（Apple Vision）でのみ有効。ON/OFFは「画面キャプチャのノイズを除去」（CLIは `--denoise`／`--no-denoise`）と共通で、OFFのときの出力は従来と完全に同じです。

**改善**
- 折り返し連結（`reflow_ocr_lines`）とラベル除去でブロック分割（縦に連続する同じ列の行のまとめ）を共有し、判定を一貫させました。

**直したところ**
- READMEの機能一覧で「画面キャプチャのノイズ除去」を**既定OFF**と誤記していたのを修正（v1.6.0で既定ONに変更済み。実際の挙動はONで、`--no-denoise`／チェックを外すとOFF）。

**原理的な限界（既知）**
- 「画面最上部・本文と別サイズ・孤立・ごく短い」行は、カテゴリ表示（国内/国際）と、稀にあるごく短い見出し（例「第一章」「速報」）を座標だけでは区別できません。この位置・形の行はラベルとして除かれることがあります（本機能の狙いは前者の除去なので許容範囲）。段落本文・句読点を含む行・一定長以上の見出しは常に残るため、記事本文の取りこぼしは起きません。気になる場合は「画面キャプチャのノイズを除去」をOFFにしてください。

## v1.6.0 — 2026-07-14（賢い区切り・使い勝手の改善）

**「画像を入れたら自動でキレイな記事テキストに」**：画像OCRでは、下の2つが自動で効くようにしました（設定不要）。
- **折り返しを座標で連結**：macOSのOCR（Apple Vision）が返す各行の位置・大きさ（外接矩形）を使い、折り返しで複数行に割れた1文を1文に戻します。見出し・箇条書き・別段落は「右端が短い」「行間が空く」ことから座標で判別し、右余白はブロック（縦に連続する同じ列）ごとに測るので、テキストだけの推測より誤連結を大幅に減らします。行末が鉤括弧「」や読点でも、折り返しなら正しく連結します。※行間が詰まった同じ幅の見出しが2本並ぶ等、座標でも区別できないケースは連結され得ます。
- **ノイズ除去を既定ON・強化**：局ロゴ・時刻・SNSハンドル・矢印などに加え、**埋め込みの英文ツイート等の外国語ブロックも除去**するようにし、既定でONにしました（`--no-denoise`／チェックを外せば従来どおり残せます）。句読点・ひらがなを含む日本語本文は必ず残します。英語主体の入力は丸ごと消えません。
- **折り返しを連結（テキスト向け・オプション）**：座標が無い入力（テキストファイル・テキスト層つきPDF）向けに、折り返しを推測で連結する簡易機能も用意。ただしテキストだけでは折り返しと別々の短い見出しを完全には区別できないため**既定OFF**（GUIの「折り返しを連結（1段組みの本文向け）」／CLIは`--smart-join`）。積極的に連結したいときは従来の「改行で途切れた文を連結（小説向け）」を使います。
- **本文の「全消去」「復元」**：抽出結果を一気に消せるボタンと、それを元に戻すボタンを追加。抽出・置換・手入力など本文が変わると「復元」は自動で無効化され、古い内容で上書きする事故を防ぎます（Ctrl/Cmd+Zでも戻せます）。
- **画像を画面のどこにでもドロップ**：ファイル一覧だけでなく、本文欄を含む画面のどこに画像・PDF・フォルダを落としても入力に追加できるようにしました。

**改善**
- 抽出オプションのチェックボックスを2段に整理し、詰め込みすぎを解消して見やすくしました。

## v1.5.0 — 2026-07-14（画面キャプチャのノイズ除去）

**新機能**
- **画面キャプチャのノイズを除去**：ニュース番組やSNS埋め込みを含む「画面キャプチャ」のOCRから、読み上げ対象でない“映像内オーバーレイ文字”（局ロゴ・時刻・日付・SNSハンドル・矢印・単独の数値など）を自動で取り除きます。判定は1行単位で保守的に行い（迷ったら残す）、句読点・鉤括弧・ひらがなを含む本文は必ず残します。既定はOFF。GUIは整形オプションの「画面キャプチャのノイズを除去」、CLIは `--denoise` で有効化できます。オフにしたときの出力は従来と完全に同じです。

## v1.4.1 — 2026-07-13（配布向けの地固め）

派手な新機能はなし。人に渡しても最初のところで詰まらない・固まらない、を目的にした安定化です。

**直したところ**
- 数値の欄（OCR解像度・話速・音高・抑揚・音量・無音）を空にしたまま実行すると、画面全体が固まって操作できなくなることがあった問題を修正。
- 前回サブモニタで閉じたあと、そのモニタを外して起動すると窓が画面の外に出て「起動しない」ように見えることがあった問題を修正（画面内に戻します）。
- 章のファイル名に日本語や空白を含む EPUB で、章がまるごと読み飛ばされることがあった問題を修正。全章読めなかったときはエラーで気づけるようにしました。
- 「連続再生（カーソル行から）」で、末尾の空行にカーソルがあると先頭から再生し直してしまう挙動を修正。
- 連続再生の最中にテキスト抽出などを始められてしまい、再生中の本文が入れ替わる不整合を防止。
- M4A/MP3 変換に失敗したとき、日本語のフォルダ名が原因でエラー表示自体が化けて落ちることがあったのを修正。

**配布まわり**
- まっさらな Windows（Python 未導入）で `setup.bat` を動かしたとき、Microsoft Store のダミー python に当たって不親切なメッセージで止まっていたのを改善。Python の入れ方（python.org / "Add python.exe to PATH"）を案内します。
- 依存ライブラリを `requirements.txt` に一本化し、バージョンに上限を付けました（将来ライブラリ側の破壊的更新で新規インストールだけ壊れるのを防ぐため）。
- アプリのバージョンをタイトルバー・`cli.py --version` に表示。不具合報告のときに版が分かります。

**後片付け・その他**
- OCR に使った一時ファイル（元文書の画像コピー・抽出テキスト）を、処理後に必ず削除するようにしました（%TEMP% に残らない）。
- CLI: `--srt` を `--combine` なしで指定したときに警告を出すように。`@話者名:` タグを GUI と同じく解釈するように（タグは読み上げず、その行だけ話者を切り替え）。
